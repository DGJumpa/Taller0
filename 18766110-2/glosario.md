1- Control de versiones (VC): El control de versiones es un sistema que registra los cambios realizados 
sobre un archivo a lo largo del tiempo, permitiendo recuperar versiones anteriores de este en caso de errores o fallos.

fuente:   https://git-scm.com/book/es/v1/Empezando-Acerca-del-control-de-versiones

2- Control de versiones distribuido (DVC): El control de versiones distribuido trata de resolver el mismo problema
que el VC, simplemente hace uso de los clientes para guardar una copia de los repositorios que el cliente haya descargado.
Así, si un servidor muere, y estos sistemas estaban colaborando a través de él, cualquiera de los repositorios de los 
clientes puede copiarse en el servidor para restaurarlo.

fuente:   https://git-scm.com/book/es/v1/Empezando-Acerca-del-control-de-versiones

3- Repositorio remoto y Repositorio local: Los repositorios remotos son versiones de un proyecto que se encuentran alojados 
en otros computadores de la red (servers) y no en nuestro propio computador. En caso de estar en nuestro propio computador, el 
Repositorio se consideraria uno local.

fuente:   https://git-scm.com/book/es/v1/Fundamentos-de-Git-Trabajando-con-repositorios-remotos

4- Copia de trabajo / Working Copy: Las Copias de trabajo se podrian considerar un repositorio local al cual se le van a aplicando los cambios.

fuente:   https://www.thomas-krenn.com/en/wiki/Git_Basic_Terms

5- Área de Preparación / Staging Area : El área de preparación es un archivo que almacena información acerca de lo que va a ir en tu próxima confirmación (commit). 
A veces se le denomina índice o index, pero se está convirtiendo en estándar el referirse a ella como el área de preparación.

https://git-scm.com/book/es/v1/Empezando-Fundamentos-de-Git

6- Preparar Cambios / Stage Changes: Hace referencia a la postulacion de una version del proyecto a pasar a ser
el proyecto principal.

fuente:   https://linuxacademy.com/blog/linux/git-terms-explained/

7- Confirmar cambios / Commit Changes: Es guardar los cambios de una version del proyecto en el repositorio local

fuente:   https://linuxacademy.com/blog/linux/git-terms-explained/

8- Commit: Es un comando para guardar versiones en el repositorio local.

fuente:   https://linuxacademy.com/blog/linux/git-terms-explained/

9- Clone: Es un comando para copiar un repositorio que está declarado como una versión
de un proyecto.

fuente:   https://linuxacademy.com/blog/linux/git-terms-explained/

10- Pull: Es un comando en dónde se presenta los cambios de un repositorio local paralelo a un repositorio remoto 
y si este es acceptado se le aplican los cambios al repositorio remoto.

fuente:   https://linuxacademy.com/blog/linux/git-terms-explained/

11- Push
	Es un comando de git que aplica los cambios de un repositorio local al remoto sin preguntar nada a diferencial del pull.

fuente:   https://linuxacademy.com/blog/linux/git-terms-explained/

12- Fetch: Es un comando de git que sirve para descargar archivos de commits de un repositorio remoto a un repositorio local.

fuente:   https://linuxacademy.com/blog/linux/git-terms-explained/

13- Merge: Es un comando de git para integrar cambios desde otra rama a la principal.

fuente:   https://linuxacademy.com/blog/linux/git-terms-explained/

14- Status: Este comando de git entrega el estado del área de inicio y del directorio de trabajo.

fuente:   https://linuxacademy.com/blog/linux/git-terms-explained/

15- Log: Este comando incluye distintas funciones para ver las diferencias entre cada commit.

fuente:   https://git-scm.com/docs/git-log

16- Checkout:  Este comando te permite navegar entre ramas creadas por una rama de git.

fuente:   https://linuxacademy.com/blog/linux/git-terms-explained/


17- Rama / Branch: Es una versión de el programa, proyecto o repositorio en el que se está trabajando, ya sea la principal u otra.

fuente:   https://linuxacademy.com/blog/linux/git-terms-explained/

18-  Etiqueta / Tag: Son etiquetas que van adjuntas a cada commit, proyecto, branch, son parte de la Documentación del proyecto.

fuente:   https://linuxacademy.com/blog/linux/git-terms-explained/
